pysh.transforms.protectnames module¶

Detects shadowings of the _ variable (used for paths), producing a syntax error when detected.

Some programing styles use _ as a way to ignore a result, which would break the DSL.

Note

At some point this transformation should not report an error but rename the references to the protected name.
